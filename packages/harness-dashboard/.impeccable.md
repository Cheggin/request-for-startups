## Design Context

### Users
Startup founders who are technical. They're running multiple autonomous AI agents building their startups. They check this dashboard while working on other things — it needs to surface the most important info instantly. Context: desktop browser, usually during working hours, often with 20+ tabs open.

### Brand Personality
**Precise, Confident, Understated.** This is a command center, not a marketing site. It should feel like a well-built instrument — every element earns its place, nothing decorative. Think Bloomberg terminal meets a Swiss railway timetable: dense where it needs to be, spacious where it doesn't.

### Aesthetic Direction
- **Tone**: Editorial/utilitarian. Not playful, not corporate. Dense information presented with clarity.
- **References**: Linear's dashboard (information density), Vercel's dashboard (typography), Stripe's charts (data visualization), Things 3 (spatial rhythm)
- **Anti-references**: Generic admin templates, Recharts default styling, "glassmorphism SaaS", hero metric layout templates
- **Theme**: Light. Founders work during the day. Tinted warm neutrals, not pure gray.

### Design Principles
1. **Information density over decoration** — every pixel communicates. No ornamental elements.
2. **Hierarchy through space, not color** — use generous whitespace to separate, tight grouping to associate. Color is accent only.
3. **Typography carries the design** — distinctive font pairing, strong size contrast, tabular numbers everywhere data appears.
4. **Real data or nothing** — never show placeholder, mock, or generated data. "No data" is more honest than fake sparklines.
5. **Zero-click answers** — the most important information (what agents are doing) visible without clicking anything.
