## Design Context

### Users
CEO/orchestrator (single user) monitoring an agent fleet while dispatching work via tmux. Glancing at the dashboard from across the room — needs to scan status in under 2 seconds. Not a deep analysis tool; this is a monitoring surface.

### Brand Personality
Precise. Quiet. Trustworthy. Like an instrument panel in a well-designed cockpit — every element earns its place, nothing decorative.

### Aesthetic Direction
- **Visual tone**: Linear/Vercel minimal — light background, monochrome palette, semantic colors only for status
- **Theme**: Light only. No dark mode.
- **Typography**: System sans-serif for labels, system monospace for data values. Two font roles, not two font families for "branding."
- **References**: Linear app, Vercel dashboard, Resend
- **Anti-references**: Terminal/htop aesthetic, NASA mission control, the retired harness-dashboard (warm OKLCH, Bricolage Grotesque, Geist fonts)
- **Color**: Cool neutrals (no warm tint). Status colors are the only saturated values: green (running), amber (idle), red (stuck), blue (info/progress).

### Design Principles
1. **Scannable from across the room** — hero numbers at 48px mono, status dots at 8px with high-contrast color
2. **Intervention-first** — stuck agents sort to top, nudge inputs always visible, not hidden behind actions
3. **Honest data** — show "No data" when unavailable, never mock or estimate
4. **Minimal means intentional** — every border, every gap, every color choice is deliberate; nothing is default
5. **Dense but not crowded** — 4-column grid, compact sparklines, tight trace log — but always with breathing room between sections
